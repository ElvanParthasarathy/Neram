**I built an entire academic management system for my college. From scratch. In 4 months.**

Here's the backstory:

At the start of this semester, I saw our timetable being shared as a photo in a WhatsApp group. It was barely readable. Someone had already reposted a cleaner version — but I thought, *why stop at a better image?*

That was December 2025. Today, I'm sharing **Neram** (நேரம் — Tamil for "Time").

**Neram is a complete multi-platform academic system** that replaces:
📱 WhatsApp timetable photos → with a live, real-time schedule viewer
📄 PDF exam schedules → with structured, searchable, date-sorted exam data
📅 Random calendar files → with OCR-powered digital academic calendar
📁 Scattered Google Drive notes → with organized, in-app study materials
📢 Missed announcements → with instant push notifications

**What I actually built (3 applications, 1 ecosystem):**

🌐 **Student Web Portal** — React 19 + Vite PWA
📱 **Student Android App** — Kotlin + Jetpack Compose (Material 3)
🛡️ **Admin Dashboard** — 16 management modules + Android wrapper app

**Key technical highlights:**
→ Real-time sync across all platforms via Firebase (changes appear in 1-2 seconds)
→ Offline-first Android app with Room Database + WorkManager background sync
→ Custom OCR engine ("Elvan Agazhi") that converts printed PDF calendars into digital data using Tesseract.js
→ 4-tier role-based access control (Super Admin → Faculty → Rep → Student)
→ macOS Sequoia-inspired glassmorphism design — 63,000+ lines of CSS
→ 304+ git commits | 906 web modules | 16+ Android Compose screens
→ Zero operating cost (Firebase + Vercel free tiers)

**The real validation?**
85 students in my class are actively using it right now. Not a prototype — a production system.

**What's next:** I've prepared a complete college handover plan. Only 4 config files need to change to deploy it institution-wide. The technical team can take it from here — I built the skeleton, the foundation, a working system. Not just an idea on paper.

This started because a timetable photo didn't look good enough.
It ended with a full-stack, multi-platform academic portal.

Sometimes the best projects come from the smallest frustrations. ✨

## 📱 SECOND COMMENT (Optional — for engagement)

```
For those asking about the OCR Calendar Engine (Elvan Agazhi):

The problem: Our college publishes the academic calendar as a printed PDF. Entering 100+ events manually is painful.

The solution: I built a client-side OCR pipeline that:
1. Uploads any academic calendar PDF
2. Renders pages using pdf.js
3. Lets you crop the relevant column
4. Extracts text using Tesseract.js (runs in browser, no server)
5. Parses dates/events using pattern matching
6. Shows a visual editor for review
7. One-click push to Firebase → live on all student devices

All processing happens in the browser. Zero server cost. Zero API fees.

This alone could be a standalone product.
```
